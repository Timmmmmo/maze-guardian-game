# 宝石塔防 - 手机简化版设计方案

## 1. 游戏设计概述

### 1.1 游戏名称与定位
- **游戏名称**：《宝石塔防·精简版》(Gem TD Lite)
- **游戏定位**：竖屏休闲塔防手游，轻度策略，碎片化时间
- **核心特色**：宝石合成系统 + 塔防建造，竖屏单手可玩

### 1.2 核心玩法描述
- 玩家在路径两侧建造塔座
- 塔座可镶嵌宝石（红/蓝/绿 3色）
- **宝石合成**：同色宝石 3 个 → 合成 1 个高级宝石（最高 3 级）
- 敌人沿固定路径移动，塔自动攻击
- 击杀敌人获得金币 → 购买更多塔/宝石

### 1.3 目标用户群体
- 15-35 岁休闲玩家
- 碎片化时间充裕（通勤、等候）
- 偏好策略/合成类游戏

## 2. 核心机制设计

### 2.1 宝石系统（简化版 - 3级）

| 属性 | 1级宝石 | 2级宝石 | 3级宝石 |
|------|---------|---------|---------|
| **外观** | 小圆 | 中圆 | 大圆+发光 |
| **攻击伤害** | 10 | 30 | 80 |
| **攻击范围** | 100px | 120px | 150px |
| **攻击速度** | 1.0s/次 | 0.8s/次 | 0.6s/次 |

#### 宝石颜色与属性

| 颜色 | 属性名称 | 效果 |
|------|----------|------|
| 🔴 红色 | 火焰宝石 | 高单体伤害，克制高血量敌人 |
| 🔵 蓝色 | 寒冰宝石 | 中等伤害 + 减速效果（敌人移速-30%） |
| 🟢 绿色 | 剧毒宝石 | 低伤害 + 持续毒伤（每秒5点，持续3秒） |

#### 合成规则
- **3 个同色同级的宝石 → 合成 1 个同色高一级宝石**
- 最高等级为 3 级，3 级宝石无法再合成
- 宝石槽位满时，新获得的宝石自动进入背包

### 2.2 塔系统

#### 基础塔设计
| 塔类型 | 建造费用 | 基础属性 | 特点 |
|--------|----------|----------|------|
| ⚔️ 剑塔 | 50 金 | 攻击10，范围100 | 基础攻击塔 |
| 🛡️ 盾塔 | 50 金 | 攻击5，范围80 | 攻速快，伤害低 |

#### 塔 + 宝石组合效果

| 组合 | 塔类型 | 宝石 | 效果描述 | 额外能力 |
|------|--------|------|----------|----------|
| 剑+红 | 剑塔 | 🔴火焰 | 火焰剑塔 | 伤害+50%，溅射周围 |
| 剑+蓝 | 剑塔 | 🔵寒冰 | 寒冰剑塔 | 伤害+30%，减速敌人 |
| 剑+绿 | 剑塔 | 🟢剧毒 | 剧毒剑塔 | 伤害+20%，附加毒伤 |
| 盾+红 | 盾塔 | 🔴火焰 | 火焰盾塔 | 攻速+50% |
| 盾+蓝 | 盾塔 | 🔵寒冰 | 寒冰盾塔 | 攻速+30%，范围+20% |
| 盾+绿 | 盾塔 | 🟢剧毒 | 剧毒盾塔 | 攻速+30%，附加毒伤 |

#### 塔的等级与升级
- 每座塔默认 1 级
- 镶嵌宝石后，塔获得宝石对应的额外属性
- 升级：花费金币提升塔的基础属性（攻击+20%，范围+10%）

### 2.3 敌人系统

#### 基础怪物类型

| 敌人类型 | 外观 | 血量 | 移速 | 抗性 | 金币奖励 |
|----------|------|------|------|------|----------|
| 👻 小幽灵 | 小幽灵 | 50 | 2.0 | 无 | 5 金 |
| 🧟 僵尸 | 普通 | 150 | 1.0 | 无 | 15 金 |
| 💀 骷髅 | 中等 | 300 | 1.5 | 减速抗性30% | 30 金 |
| 👹 巨魔 | 大只 | 800 | 0.8 | 毒伤抗性50% | 80 金 |
| 🐉 巨龙 | BOSS | 3000 | 1.0 | 全抗性30% | 300 金 |

#### 怪物属性说明
- **血量**：生命值，归零则死亡
- **移速**：像素/帧，越低越慢
- **抗性**：对特定效果减免（减速效果减少30%等）
- **金币奖励**：击杀后玩家获得

### 2.4 波次系统

| 波次 | 敌人数 | 怪物类型 | 间隔 | 难度系数 |
|------|--------|----------|------|----------|
| 1-3 | 5-8 | 小幽灵 | 2s | 1.0 |
| 4-6 | 8-12 | 小幽灵+僵尸 | 1.8s | 1.2 |
| 7-9 | 10-15 | 僵尸+骷髅 | 1.5s | 1.5 |
| 10 | BOSS波 | 巨魔+巨龙 | 2s | 2.0 |

- 每波间隔 10 秒
- 难度递增：敌人血量 +10%/波
- 每通过 5 波，恢复一次生命值（基地血量回复20%）

## 3. 手机适配设计

### 3.1 竖屏操作优化
- **屏幕比例**：适配 9:16 ~ 9:20
- **单手操作区域**：屏幕下半部分为操作区
- **触控热区**：宝石拖拽区位于底部 30% 屏幕

### 3.2 UI 布局

```
┌──────────────────────┐
│   波次 / 基地血量 / 金币    │ ← 顶部状态栏 (10%)
├──────────────────────┤
│                      │
│      敌人移动路径         │ ← 游戏主区域 (60%)
│      塔建造位置           │
│                      │
├──────────────────────┤
│  [宝石槽1] [宝石槽2] [宝石槽3] │ ← 宝石槽 (15%)
├──────────────────────┤
│  [购买剑塔] [购买盾塔] [升级]  │ ← 功能按钮 (15%)
└──────────────────────┘
```

### 3.3 操作方式
- **建造塔**：点击"购买塔"按钮 → 点击地图上的建造点
- **镶嵌宝石**：从底部宝石槽拖拽到塔上
- **升级塔**：点击塔 → 点击升级按钮
- **出售塔**：长按塔 → 显示出售按钮

## 4. 商业化设计

### 4.1 体力系统
- **体力上限**：10 点
- **每局消耗**：1 点体力
- **体力恢复**：每 5 分钟恢复 1 点
- **体力购买**：10 元 = 10 体力（可选）

### 4.2 变现点
- **皮肤系统**：塔和宝石的外观皮肤（特效皮肤）
- **宝石礼包**：首充 6 元送 100 个随机宝石
- **体力购买**：快速恢复体力
- **无尽模式**：通关后解锁，无尽波次挑战
- **排行榜**：无尽模式波次排名

## 5. 技术架构

### 5.1 引擎选择
- **首选**：Cocos Creator 3.x（支持 H5 + 原生双端）
- **备选**：Phaser.js（轻量 H5 原型）
- **原型阶段**：纯 HTML5 + Canvas + JavaScript

### 5.2 核心模块划分

```
┌─────────────────────────────────────────┐
│                游戏主循环                  │
│  (requestAnimationFrame, 60FPS)          │
├──────────┬──────────┬──────────┬─────────┤
│ 敌人模块  │  塔模块   │ 宝石模块  │ UI模块   │
│ Enemy    │  Tower   │  Gem     │ UI      │
│ - 生成    │ - 建造    │ - 合成   │ - HUD   │
│ - 移动    │ - 攻击    │ - 镶嵌   │ - 按钮  │
│ - 死亡    │ - 升级    │ - 属性   │ - 弹窗  │
├──────────┴──────────┴──────────┴─────────┤
│               资源管理                    │
│  - 图片加载 / 音频管理 / 存档系统          │
└─────────────────────────────────────────┘
```

### 5.3 数据结构

```javascript
// 宝石
Gem = { id, color, level, x, y }

// 塔
Tower = { id, type, x, y, level, gem, attack, range, speed }

// 敌人
Enemy = { id, type, hp, maxHp, x, y, speed, pathIndex }

// 波次
Wave = { waveNumber, enemies[], spawnInterval }
```
