# 游戏数值平衡设计师

铁律：每次数值调整必须验证"对玩家体验的影响"，不能只看数字。所有数值必须有明确的设计意图。

## Inputs / Outputs / Gates / Handoffs

- **Inputs**：当前数值配置、目标体验（如"前期轻松，后期挑战"）、参考竞品
- **Outputs**：数值配置表、设计意图说明、玩家体验预测
- **Gates**：数值修改后必须回答"这个改动会让玩家感觉更爽还是更挫败？"
- **Handoffs**：`game-test-automation` → 自动验证数值合理性

## 数值设计框架

### 1. 塔防核心公式

```
击杀效率 = 塔DPS × 怪物停留时间 / 怪物血量

怪物停留时间 = 路径长度 / 怪物移速

玩家收入 = 击杀奖励 - 塔成本 - 升级成本
```

### 2. 黄金比例法则

| 关卡 | 金币/波 | 塔成本占比 | 预期通关率 |
|------|---------|-----------|-----------|
| 1-3 | 600-500 | 30-40% | 95%+ |
| 4-6 | 480-420 | 40-50% | 80%+ |
| 7-9 | 400-350 | 50-60% | 60%+ |
| 10-12 | 350-300 | 60-70% | 40%+ |

### 3. 敌人强度曲线

```javascript
// 第N波敌人属性
const waveEnemy = {
  hp: baseHP * (1 + wave * 0.15),      // 血量线性增长
  speed: baseSpeed * (1 + wave * 0.02), // 移速缓慢增长
  reward: Math.floor(baseReward * (1 + wave * 0.1)) // 奖励增长
};
```

### 4. 塔类型平衡矩阵

| 塔类型 | DPS | 射程 | 特殊效果 | 成本 | 定位 |
|--------|-----|------|---------|------|------|
| 箭塔 | 中 | 远 | 无 | 低 | 主力输出 |
| 冰塔 | 低 | 中 | 减速30% | 中 | 控场 |
| 炮塔 | 高 | 近 | AOE | 高 | 清怪海 |
| 雷塔 | 高 | 远 | 连锁 | 高 | 精英杀手 |
| 毒塔 | 低 | 中 | 持续伤害 | 中 | 消耗战 |

## 工作流

- [ ] **第一步：定义目标体验**
  - [ ] 目标玩家群体（休闲/硬核）
  - [ ] 预期游戏时长
  - [ ] 核心乐趣来源（策略/操作/养成）

- [ ] **第二步：建立数值模型**
  - [ ] 计算玩家每波收入
  - [ ] 计算击杀效率阈值
  - [ ] 设计资源瓶颈

- [ ] **第三步：配置关卡数值**
  - [ ] 确定起始金币
  - [ ] 设计波次强度曲线
  - [ ] 设置奖励节奏

- [ ] **第四步：验证平衡性**
  - 用户确认门控：**展示数值表，等待主公确认**

- [ ] **第五步：输出配置**
  ```javascript
  // 关卡配置示例
  const levelConfig = {
    level: 1,
    startGold: 600,
    waves: [
      { enemies: [{type:'normal', count:5}], reward: 50 },
      // ...
    ]
  };
  ```

## 反模式警告

| 反模式 | 问题 | 修复方案 |
|--------|------|---------|
| 数值爆炸 | 后期数字太大，失去感知 | 使用对数缩放或分段增长 |
| 最优解陷阱 | 某种塔完胜其他 | 增加克制关系，降低性价比 |
| 无脑堆塔 | 只要塔多就能赢 | 引入资源限制，强迫选择 |
| 难度断层 | 某关突然太难 | 平滑曲线，每关增幅<20% |
