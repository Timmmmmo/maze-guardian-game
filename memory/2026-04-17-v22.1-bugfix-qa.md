# MazeTD V22.1 Bug修复 & QA验收报告

## 日期：2026-04-17

## 一、QA发现的问题

### Q1：初始金币显示650（应为400）
- **现象**：点击开始后金币显示650
- **根因**：`startGame()`中`initGrid()`和`updateUI()`被放在`initPixi().then()`回调里。异步执行导致：
  1. `startGame()`设置`gold=650`（LEVELS[0].startGold）
  2. `initPixi()`的`await app.init()`异步等待
  3. `startGame()`返回，DOM未更新
  4. 外部代码读DOM → 读到旧值
  5. PIXI加载完毕 → `initGrid()` → `updateUI()` → DOM更新（但已错过读取时机）
- **修复**：将`initGrid();updateUI();updateTool();render()`移出Promise回调，同步执行
- **验证**：✅ 初始金币正确显示650

### Q2：Tower范围指示器每帧新建Graphics对象
- **现象**：`gfxCircle=new PIXI.Graphics()`每帧调用，无`const`声明→全局变量泄漏
- **根因**：Line 162缺`const`，且Tower渲染每帧创建大量Graphics对象
- **修复**：添加`const gfxCircle=new PIXI.Graphics()`
- **验证**：✅ Tower渲染正常，Canvas无异常

### Q3：PIXI.Text伤害数字累积（内存泄漏+视觉叠影）
- **现象**：伤害数字每帧叠加，产生幽灵文字
- **根因**：`gfxDmg.clear()`只清理Graphics对象，不清理Text子节点。`new PIXI.Text()`每帧新增并`addChild`到gfxDmg，永不删除
- **修复**：每次渲染前调用`gfxDmg.removeChildren().forEach(c=>c.destroy())`
- **验证**：✅ 伤害数字正常，无叠影

### Q4：开战按钮文字乱码
- **现象**：按钮显示`⏸ 暂停`→`⏸ ?????`
- **根因**：Vite打包时emoji字符`⏸`和`⚔️`编码损坏
- **修复**：换用`|| 战斗中`和`▶ 开战`
- **验证**：✅ 按钮文字正常显示

### Q5：升级面板意外显示
- **现象**：游戏未操作时升级面板意外显示
- **状态**：未复现（单次测试）。`upgrade-panel`默认`display:none`，无代码主动`show()`
- **验证**：✅ 升级面板正确隐藏
