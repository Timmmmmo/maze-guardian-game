# HappyBricks 产品经理团队讨论记录

**时间**: 2025-07-25
**任务**: 围绕"如何让HappyBricks更好玩"进行产品经理内部讨论

## 游戏当前状态（V5.2）
- 核心玩法：弹球物理消除石块，360°瞄准射击
- 题材：父子被困石圈中央，清石营救
- 系统：压力条、连击(x2/x3/x5/x8)、4种特殊石块（金币/炸弹/时间/分裂）
- 5固定关卡+随机生成
- 土棕色复古风格，Web Audio API 9种音效

## 10轮讨论摘要

| 轮次 | 议题 | 结论 |
|------|------|------|
| R1 | 核心爽感 | 引入连锁引爆机制，单次操作产生不可预测的连锁爽感 |
| R2 | 战略深度 | 增加结构坍塌物理效果，瞄准位置比角度更有策略意义 |
| R3 | 叙事利用 | 父子动态对白+表情变化，鼓励性反馈增强情感，不惩罚玩家 |
| R4 | 重玩价值 | 第5关后进入"变异无尽模式"，每5关叠加规则变体 |
| R5 | 阶段目标 | 引入阶段守护石，击破触发关卡事件和大量奖励 |
| R6 | 差异化 | 锚点="物理坍塌+营救叙事"，所有功能围绕此展开 |
| R7 | 付费设计 | 主题关卡包+装饰性皮肤，绝不卖数值 |
| R8 | 留存机制 | 强化"差一点"体验，失败时展示明确进度提示 |
| R9 | 连击升级 | 双轨制：速度连击+技巧连击，技巧连击收益更高 |
| R10 | 音视配合 | 战斗用递进音效+震屏，营救成功用情感反差音乐切换 |

## 📋 产品一致结论

### 核心问题诊断
当前游戏"消除本身不爽、策略选择太浅、营救叙事只是皮"。玩家打两局就觉得"就那样"，因为单次操作缺乏连锁爆发感，每局体验高度重复，父子主题没有产生情感粘性。

### 改进方向优先级

**P0（下一版本必须做）**：物理坍塌系统+连锁引爆机制 + 音视反馈强化（递进连击音效+震屏）

**P1（下两个版本要做）**：变异无尽模式（每5关叠加规则变体）+ 阶段守护石 + 父子动态对白

**P2（未来探索）**：主题关卡包付费 + 每日挑战排行榜 + "差一点"失败提示系统 + 技巧连击双轨制

### 一句话总结
**把"弹球消除"变成"物理连锁营救"——每一次射击都可能引发坍塌与连环爆炸，每一次失败都只差一步，让玩家为那一击的爽感和那一步的执念反复回来。**
